# payment-service

Core service of the payment gateway responsible for payment processing and lifecycle management.

## Overview

`payment-service` owns the payment aggregate and its state machine. It orchestrates the full payment flow: initialization, confirmation, cancellation.

## Architecture

The service follows a layered internal structure (domain / application / infrastructure) and is designed to scale horizontally. It is part of the [payment-gateway](https://github.com/sergeevoa/payment-gateway) microservice application.

---

## Database Schema

> <span style="color:red">*</span> — required field (NOT NULL, no default)

### `payments`

Stores the full lifecycle of each payment initiated by a merchant.

| Field | Type / Default | Constraints | Description |
|---|---|---|---|
| <span style="color:red">*</span>`payment_id` | `uuid` | PK | Unique payment identifier generated by the gateway |
| <span style="color:red">*</span>`terminal_id` | `varchar(20)` | NOT NULL | Terminal that initiated the payment |
| <span style="color:red">*</span>`order_id` | `varchar(36)` | NOT NULL | Order identifier in the merchant's system |
| `customer_id` | `varchar(36)` | — | Customer identifier in the merchant's system |
| <span style="color:red">*</span>`amount` | `bigint` | NOT NULL | Payment amount in minor currency units (e.g. kopecks) |
| <span style="color:red">*</span>`status` | `varchar(20)` | NOT NULL | Payment status: `NEW`, `AUTHORIZED`, `CONFIRMED`, `REVERSED`, `REFUNDED`, `PARTIAL_REFUNDED`, `REJECTED` |
| `description` | `varchar(140)` | — | Order description shown on the payment form |
| <span style="color:red">*</span>`is_recurrent` | `boolean` / default `false` | NOT NULL | Whether card credentials should be saved for future recurring payments |
| `rebill_id` | `uuid` | — | Recurring payment identifier, assigned after a successful recurrent payment |
| `card_id` | `uuid` | — | Bound card used for this payment, if applicable |
| <span style="color:red">*</span>`language` | `varchar(2)` / default `'en'` | NOT NULL, CHECK (`ru`, `en`) | Language of the payment form |
| `payment_url` | `text` | — | URL of the payment form to redirect the customer to |
| `notification_url` | `text` | — | Webhook URL for payment status change notifications (per-request override) |
| `success_url` | `text` | — | Redirect URL after successful payment |
| `fail_url` | `text` | — | Redirect URL after failed payment |
| `redirect_due_date` | `timestamptz` | — | Expiry date and time of the payment link (UTC) |
| `send_email` | `boolean` | — | Whether to send a payment receipt to the customer's email |
| `info_email` | `varchar(255)` | — | Customer email for the receipt |
| <span style="color:red">*</span>`created_at` | `timestamptz` / default `now()` | NOT NULL | Timestamp when the payment record was created |
| <span style="color:red">*</span>`updated_at` | `timestamptz` / default `now()` | NOT NULL | Timestamp of the last status update |

---

### `operations`

Stores the bank-confirmed operation details for a payment. Has a strict one-to-one relationship with `payments`.

| Field | Type / Default | Constraints | Description |
|---|---|---|---|
| <span style="color:red">*</span>`operation_id` | `uuid` | PK | Unique operation identifier generated by the gateway |
| <span style="color:red">*</span>`payment_id` | `uuid` | NOT NULL, FK → `payments.payment_id`, UNIQUE | Payment this operation belongs to |
| <span style="color:red">*</span>`status` | `varchar(20)` | NOT NULL | Operation status (e.g. `CONFIRMED`) |
| <span style="color:red">*</span>`amount` | `bigint` | NOT NULL | Authorized amount in minor currency units |
| `auth_code` | `varchar(20)` | — | Authorization code issued by the issuing bank |
| `auth_date` | `timestamptz` | — | Date and time of bank authorization |
| <span style="color:red">*</span>`created_at` | `timestamptz` / default `now()` | NOT NULL | Timestamp when the operation record was created |
